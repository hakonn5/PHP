# Лабораторная работа №6. Обработка и валидация форм

## Тема проекта: Трекер привычек

**Модель данных (Habit):**

1. `id` (string) — уникальный идентификатор записи.
2. `name` (string) — название привычки.
3. `description` (text) — подробное описание или мотивация.
4. `start_date` (date) — дата начала трекинга.
5. `category` (enum) — категория (Здоровье, Обучение, Спорт, Финансы).
6. `is_daily` (boolean/checkbox) — требуется ли ежедневное выполнение (Да/Нет).
7. `created_at` (date) — системная дата создания записи.

## Цель работы

Освоить основные принципы работы с HTML-формами в PHP, отправку данных на сервер, их валидацию и сохранение в файл.

## Описание реализации

### 1. Создание HTML-формы
Создана форма (файл `index.php`) для добавления новой привычки. Метод отправки формы — `POST`. Добавлена клиентская валидация с использованием HTML5 атрибутов (`required`, `minlength="3"`).

### 2. Обработка данных на сервере и валидация
На стороне сервера данные принимаются из глобального массива `$_POST`.
Для повышения надежности введена проверка и валидация входных данных, а также защита от XSS с помощью `htmlspecialchars()`.

### 3. Задание 1. Добавление интерфейса на 10 баллов
Выполнено дополнительное задание (на 10 баллов). Реализована система валидации на основе интерфейса `ValidatorInterface`.

Файл `ValidatorInterface.php`:
```php
<?php
declare(strict_types=1);

/**
 * Интерфейс для всех правил валидации.
 * Позволяет унифицировать проверку данных.
 */
interface ValidatorInterface
{
    /**
     * Проверяет значение.
     * @param mixed $value Значение из формы
     * @return string|null Сообщение об ошибке или null, если ошибок нет
     */
    public function validate(mixed $value): ?string;
}
```

Конкретные валидаторы реализованы в `Validators.php` (например, `RequiredValidator` и `MinLengthValidator`). Они вызываются при обработке формы для проверки имени, описания и даты.

### 4. Сохранение и вывод данных
Успешно прошедшие валидацию данные собираются в ассоциативный массив и сохраняются в файл `data.json` в формате JSON.
На странице выводится HTML-таблица со списком всех сохраненных привычек. Добавлена возможность сортировки таблицы по названию, категории или дате создания с помощью GET-параметров (например, `?sort=name`).

## Контрольные вопросы

**1. Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?**
В протоколе HTTP существует множество методов отправки данных (GET, POST, PUT, DELETE, PATCH и др.). Однако **HTML-формы** нативно поддерживают только два:
- **GET** — данные передаются открыто прямо в URL-строке (например, `?name=Ivan`). Подходит для безопасных поисковых запросов и фильтрации.
- **POST** — данные передаются скрыто в теле HTTP-запроса (body). Используется для передачи конфиденциальных данных и больших объемов текста.

**2. Какие глобальные переменные используются для доступа к данным формы в PHP?**
- `$_GET` — для данных, переданных методом GET.
- `$_POST` — для данных, переданных методом POST.
- `$_FILES` — для загруженных файлов.
- `$_REQUEST` — объединяет в себе данные из `$_GET`, `$_POST` и `$_COOKIE` (использовать не рекомендуется).

**3. Как обеспечить безопасность при обработке данных из формы (например, защититься от XSS)?**
- **Защита от XSS:** Всегда использовать функцию `htmlspecialchars()` при выводе данных в HTML.
- **Валидация на сервере:** Обязательно проверять типы и значения данных в PHP, так как клиентскую валидацию легко обойти.
- **Защита от CSRF:** Использовать уникальные токены, генерируемые сервером для каждой сессии формы.
